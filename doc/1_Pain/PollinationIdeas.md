# Pollination Ideas (異分野からの着想)

SmartIMEを「単なる文字変換器」から「思考のパートナー」へ進化させるための、異分野からの技術・思想的ポリネーション。

## 1. From ASR (音声認識): 「事後最適化」の徹底
- **LID (Language Identification)**: 音声認識が「今話されているのが何語か」を瞬時に判定するように、キーボード打鍵の「リズム」や「文字分布」から言語・文脈を判定。
- **Lattice-based Processing**: 決定論的に一つの変換結果に絞るのではなく、複数の可能性（ラティス）を保持し、文の後半が打たれた時点で前半の変換を「こっそり」最適化する。

## 2. From Gaming UX: 「入力バッファリング」と「コンボ」
- **Input Buffering**: 格闘ゲームが「先行入力」を受け付けるように、OSの処理待ちが発生しても打鍵をバッファし、思考を止めない。
- **Command Combos**: 特定のローマ字シーケンスを「コマンド（マクロ）」として扱い、自然文の中にショートカットをシームレスに埋め込む（例: `,,t` -> 現在時刻）。

## 3. From IDE/Developer Tools: 「人間用LSP (Language Server Protocol)」
- **Semantic Analysis**: コードの静的解析のように、自然文の「論理的矛盾」や「接続詞のねじれ」をバックグラウンドで検知し、変換候補として修正案を提示。
- **Snippet Expansion**: プログラミングのコード片呼び出しと同じ感覚で、日常の定型文や専門用語を「不完全な入力」から補完。

## 4. From Cognitive Science: 「モードレス」な認知設計
- **Cognitive Friction Removal**: 人間が「モード」を切り替える際の数ミリ秒の脳のスイッチングコストをゼロにする。
- **Habit Compression**: ユーザー特有の「打ち間違い」や「省略癖」を、エラーではなく「自分専用のショートカット」として学習・圧縮する。

## 5. From Linguistics: 「ローマ字のインターリングア（中間言語）化」
- **Phonetic Bridge**: ローマ字を単なる「かなへの入力手段」ではなく、日本語・英語・コードを等しく扱うための「共通抽象レイヤー」として定義。
- **Multilingual Fluidity**: ローマ字で打ち始めたものが、文末の助詞で日本語になり、ハイフンでCLIコマンドに分岐する「流体のような入力」。
