# Local SLM Strategy (局所言語モデル戦略)

SmartIMEを「状態（State）」から「確率（Probability）」のエンジンへ変える核心は、デバイス内で完結する超軽量な **Local SLM (Small Language Model)** の構築にあります。その具体的な実装・運用計画を整理します。

## 1. 目的: 「言語らしさ」の瞬時判別
従来のIMEが「辞書との一致」を見るのに対し、Local SLMは「並びの確率的自然さ」を評価します。
- **入力**: 未確定のローマ字ストリーム（例: `gitcom` や `kyouhaam`）
- **出力**: 意図ラベル（CLI / JP_Natural / EN_Natural）と、その確信度スコア。

## 2. アーキテクチャ: 多層評価モデル
単一の巨大なモデルではなく、計算負荷の低い順に多層的に評価を行うことで、一瞬の遅延（10ms以内）を実現します。

### 第一層: 構造・記号フィルタ (Rule-based / Regex)
- ハイフン、スラッシュ、記号比率、特定のコマンド接頭辞を検知。
- **役割**: 明らかな CLI / 英語 を高速にバイパスする。

### 第二層: 文字分布・音韻モデル (Lightweight N-gram SLM)
- ローマ字の並びが「日本語のかな構造」として成立する確率を計算。
- **特徴**: 母音の出現パターン、助詞（`ha`, `ga`, `wo`）や文末（`desu`, `masu`）の音節並び。
- **役割**: 自然文らしさを 0.0〜1.0 の確率値で定量化する。

### 第三層: 意味・文脈モデル (Quantized Tiny Transformer)
- 30MB〜50MBに量子化された極小モデル。
- **役割**: 前後の確定済み文章から「次に何が打たれるべきか」を推論し、曖昧な入力の補正候補を生成する。

## 3. 確率的判断のプロセス
1. **Streaming Input**: 入力される1打鍵ごとにバッファを更新。
2. **Real-time Scoring**: バッファ末尾N文字から「日本語らしさ」の確率を常に更新し続ける。
3. **Commit Trigger**: Enter または フォーカスアウト時に、最も確率の高い意図に従って一括変換（または原文維持）を実行。
4. **Consistency check**: 文の途中で確率が極端に変動した場合（例: 日本語を打っていたのに途中で `git` と打った）、その地点を境界線として分割処理を行う。

## 4. 学習と適応 (Habit Compression)
- **Local Learning**: ユーザーによる「変換のやり直し」を「負のフィードバック」として学習し、個人の癖に合わせた確率マップをローカルで更新。
- **Context Prior**: 直前の履歴がターミナルなら CLI の確率に、Slackなら自然文の確率に下駄を履かせる（Priorの動的調整）。

## 5. 実装ターゲット
- **推論レイテンシ**: 10ms 未満
- **メモリ占有**: 20MB 〜 50MB
- **エンジン**: Rust による SIMD 最適化。
