# Dictionary Architecture (辞書システム)

SmartIMEにおける辞書は、静的なデータのリストではなく、リアルタイムな推論を支える **高性能な検索・記憶エンジン** です。判別モデル (Discriminator) と処理モデル (Processor) の双方から呼び出される、システムの心臓部を定義します。

## 1. 階層化辞書構造 (Layered Storage)
効率的な検索と個人適応を両立するため、辞書を三つの層に分離します。

| レイヤー | 役割 | 更新頻度 | ストレージ |
| :--- | :--- | :--- | :--- |
| **System Layer** | 一般語彙、助詞、基本コマンド。 | リリース時のみ | 読み取り専用バイナリ |
| **Domain Layer** | 専門用語（医学、IT）、特定プロジェクト。 | 設定時 | 外部プラグイン形式 |
| **Personal Delta** | ユーザー独自の変換、修正済み誤字の癖。 | 随時（リアルタイム） | ローカルDB (SQLite/Key-Value) |

## 2. 実装技術 (Technical Internals)
極限までレイテンシ（<10ms）を削るため、以下の技術を採用します。

- **Zero-Copy Memory Mapping**: ディスク上のバイナリ辞書を `mmap` し、デシリアライズなしで直接検索。
- **Double-Array Trie**: 前方一致検索を極めて高速（O(k)）に行い、入力中のリアルタイム補完を実現。
- **Probabilistic Bloom Filters**: 辞書に存在しない単語を瞬時に特定し、重い解析エンジン（MeCab等）への無駄なリクエストを遮断する。

## 3. 専門辞書の連携
意図判定の結果に基づき、使用する専門辞書を瞬時に切り替えます。
- **CLI Dictionary**: `git`, `npm`, `cd` などのコマンド名、およびその主要なオプション（`-m`, `--version`）。
- **Phonetic Bridge Dictionary**: ローマ字の綴りから語彙の存在確率（JPらしさ）を逆引きするためのスコア辞書。
- **Correction Map**: 打鍵位置と音韻に基づき、誤入力 `konnitiha` を `こんにちは` へとマッピングする辞書。

## 4. パーソナライズ (Habit Compression)
ユーザーの「癖」を学習し、辞書の重み付けを動的に変更します。
- **Trie への動的挿入**: ユーザーが新規作成した単語や、頻繁に使う略語を Personal Delta 層の Trie へ即座に反映。
- **習慣圧縮**: 直近 N 回の確定履歴に基づき、同じ頭文字から始まる複数の候補の順位を入れ替える。

## 6. コーパスの新陳代謝 (Corpus Metabolism)
辞書とモデルは固定されたものではなく、常に「新陳代謝」を繰り返します。
- **短期記憶 (Trie)**: ユーザーの最新の入力や修正を即座に Trie 構造へ反映（即時性）。
- **長期記憶 (Refined Model)**: 蓄積された履歴から確率分布を再計算し、Local SLM の重みを定期的にバックグラウンドで更新（定着）。
- **不要語のパージ**: 使用頻度が極端に低い個人語彙を自動的に整理し、辞書の肥大化を防ぐ。

## 5. オフライン整合性とプライバシー
- 辞書の同期は明示的なユーザー操作時のみ実行。
- Personal Delta は暗号化され、デバイス外に送信されることは決してありません。
