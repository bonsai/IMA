# Adr0019HighPerformanceAndPersonalizationOptimization

## Status
Proposed

## Context
SmartIMEは完全オフラインかつ10ms以下の超低遅延を目標としている。これまでの設計（Adr0001〜Adr0018）で基本的な構造は定義されたが、クロスプラットフォーム間での実装の重複、ユーザー固有の入力習慣への適応速度、および極限の実行速度において、さらなる最適化の余地がある。特に、ローマ字入力の即時補正（Typo Tolerance）において、CPUリソースを最小限に抑えつつ「打鍵に完全に追従する」体験を実現する必要がある。

## Decision
以下の3つの技術的アプローチを統合し、パフォーマンスとパーソナライゼーションを極限まで高める：

1.  **Personal Deltaの動的同期とパーソナライズ強化**:
    - ユーザー辞書および学習履歴（Personal Delta）を、バックグラウンドでリアルタイムに近い頻度で動的同期する。
    - 単なる語彙追加だけでなく、コンテキストに応じた確率重みの調整をPersonal Delta層で優先的に適用し、個人の癖への適応速度を最大化する。

2.  **Rustによるクロスプラットフォーム抽象レイヤーの統一**:
    - Windows (TSF)、Android (NDK/JNI)、Browser (WASM) の各プラットフォーム固有のアダプター層を最小化し、ビジネスロジックのほぼ全てを共通のRust Coreで抽象化・実装する。
    - 各プラットフォーム向けに最適化されたバインディングを生成することで、ロジックの二重実装を排除し、プラットフォームを問わず同一の爆速な体験を提供する。

3.  **下層SIMDおよびmmapによる極限の高速化**:
    - ローマ字入力の即時補正や検索処理において、SIMD（Single Instruction Multiple Data）命令を活用し、複数文字の並列処理を行う。
    - Adr0017で導入したmmapをさらに徹底し、辞書検索だけでなく、一時的なバッファやPersonal Deltaの読み書きにおいてもゼロコピー・アクセスを追求する。これにより、メモリ帯域のボトルネックを解消し、遅延を理論上の限界まで削減する。

## Rationale
- **Personal Delta**: ユーザーが「自分のIMEだ」と感じるための鍵は、修正が即座に反映される適応性にある。
- **Rust統一レイヤー**: 開発効率の向上だけでなく、メモリ管理やスレッド制御をRustで一元化することで、全プラットフォームで最高のパフォーマンスを保証できる。
- **SIMD/mmap**: 10msの壁を越え、0msに近い「体感遅延ゼロ」を達成するには、高レイヤの最適化だけでなく、ハードウェアの能力を引き出す低レイヤの命令セット活用が不可欠である。

## Consequences
- 各プラットフォーム（特にAndroid NDKやWASM）において、SIMD命令の互換性チェックが必要になる。
- Rust Coreの抽象化が進むことで、プラットフォーム固有の特殊な挙動（OS標準の予測変換連携など）のハンドリングが複雑になる可能性がある。
- Personal Deltaの動的同期により、書き込み時の競合制御（Lock-freeなデータ構造など）が重要になる。
